# Ekam MVP Frontend

Standalone web frontend for the student learning app. It reads content from the **generated_resources** folder (same structure as the Android app) and provides:

- **Dashboard** – Welcome and progress placeholder
- **Subjects** – List of subjects (Science, English) with links to chapters
- **Study table** – All chapters across subjects with resource counts
- **Chapter view** – Per-chapter resources: Summary, Study Guide, Mind Map, Flashcards, Learning Path, Detailed Notes, Podcast Script, YouTube Links, Question Bank, Course Info

## Setup

From the `mvp_frontend` directory:

```bash
npm install
```

Before first run (and whenever `generated_resources` changes), copy resources and generate the manifest:

```bash
npm run copy-resources
npm run generate-manifest
```

Or rely on the automatic steps:

- `npm run dev` runs `predev`: copy-resources + generate-manifest, then starts the dev server.
- `npm run build` runs `prebuild`: same, then builds for production.

## Run

```bash
npm run dev
```

Open http://localhost:5173. Use the bottom navigation: Dashboard, Subjects, Study Table.

## Structure (aligned with mvp_android)

| Android | Frontend |
|--------|----------|
| StudentActivity (bottom nav) | `Layout` + React Router |
| DashboardFragment | `pages/Dashboard` |
| SubjectsFragment | `pages/Subjects` |
| StudyTableFragment | `pages/StudyTable` |
| ChapterListFragment | `pages/ChapterList` |
| ResourceTabsFragment | `pages/ResourceTabs` |
| MarkdownFragment | `components/MarkdownView` |
| (WebView mermaid) | `components/MermaidView` |
| FlashcardsFragment | `components/FlashcardsView` |
| ContentRepository | `data/contentRepository` + manifest from `generated_resources` |

Content is loaded from `public/generated_resources/` (populated by `copy-resources` from `../generated_resources`). The manifest at `public/generated_resources/manifest.json` is generated by `generate-manifest` and lists subjects and chapters.
